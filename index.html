Dr. B, I know what you've done...

First things first, the whole reason you had an animation was because you were updating data. You (probably) did this by rebinding new values to existing elements. If you were using rectangles, this might've looked like this: svg.selectAll("rect")
.data(dataset);

Now as for your actual animations, it was pretty simple really. All you needed to get an animation after you successfully changed the data was one line of code:
.transition()
This one line by itself gives you a decent animation from one data set to another. Of course, you may have added more code than that to control for timing. For that, you used this line of code: duration().

The last fancy part of your animations was the hover effects you had on the graph. Murray says there are two steps to make a visualization interactive:
1. Binding event listeners
2. Defining the behavior
You specifically used a hover effect that highlighted a bar on the chart. This means your code probably looked like this: .on("mouseover", function(){...});
if I'm remembering correctly, you set the color of the bar to orange. This means that inside your function, you set the fill attribute of the rectangle to orange. Your code would've looked like this: .on("mouseover", function (){
d3.select(this)
.attr("fill", "orange");
}
The coolest thing you used above was the word this. This was a simple way of selecting the rect that just got hovered over to change color.
Finally, you probably had an event listener for when the rectangle was no longer being hovered over so that it would change back to its normal color. This could've been achieved with mouseout and then setting the fill attribute to whatever it was.
